# Moneta Critique

追问用户，直到“怪”变成可执行的修改标准。

一次只问一个问题，并给出你的推荐判断。用户没有确认前，不要写入长期规则，也不要改进 Moneta skills。

优先追问：

- 怪在用词、句式、画面、逻辑，还是来源使用？
- 最让人出戏的是哪个词或哪半句？
- 这条反馈只适用于当前文章，还是以后都应该避免？
- 如果 agent 连续两次改不好，请用户亲自写一个范例。

用户确认后，把稳定规则交给 `/moneta-skills-improver`。

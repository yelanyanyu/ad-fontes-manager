# Use electron-updater for desktop Software Update

**Status**: accepted

The desktop Software Update flow will use `electron-updater` with a GitHub provider feed for `yelanyanyu/ad-fontes-manager`, using metadata generated by the Electron packaging/release process instead of a custom GitHub Releases API downloader. This keeps version discovery, platform asset selection, download progress, cancellation, and user-confirmed installation on the standard `electron-builder` path, while avoiding the platform-specific risks of manually downloading and running installers.

The first implementation targets Windows x64 NSIS-installed builds only. Portable Windows builds and macOS DMG update behavior are deferred until their installation semantics can be tested directly.

## Considered Options

### Custom GitHub Releases downloader

Rejected because it would require the app to implement asset matching, download progress, cancellation, file integrity handling, Windows/macOS installer launch behavior, and app shutdown sequencing itself. Those responsibilities are easy to get subtly wrong and duplicate what `electron-updater` already provides.

### Silent or quit-time installation

Rejected because installation must be an explicit user action. The app may download an update automatically, but it must not install automatically when the app quits or run an installer without the user choosing **Install Update**.

### Initial unsigned Windows updates

Accepted as a temporary distribution constraint because the project does not currently have a Windows code signing certificate. The first Windows Installer Update may rely on the official GitHub Release, HTTPS transport, and `electron-updater` metadata/hash checks, with installation still requiring explicit user confirmation. When a signing certificate is available, Windows update code signature verification should be enabled.

The updater implementation should keep an explicit TODO/extension point for future Windows code signature verification rather than burying the temporary unsigned behavior in incidental configuration.

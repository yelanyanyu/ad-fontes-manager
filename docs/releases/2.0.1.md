## [2.0.1] - 2026-05-23

### 新增

- **桌面自动更新**：新增 Electron 桌面端更新检查、Release Notes 展示、跳过版本、下载完成后安装确认等完整链路。
- **自定义桌面标题栏**：Windows 桌面端改为自定义窗口标题栏，更新入口常驻在窗口右上角，发现新版本后可直接查看更新细则。
- **桌面发布工作流**：新增 GitHub Actions 一键发布桌面安装包，自动上传 Windows / macOS 安装包、blockmap 和 update feed。
- **统一默认配置**：新增 Default App Configuration，集中管理设置页默认值；新用户默认并发池为 5，并预置 DeepSeek 快速 / 均衡 / 专家模型配置。
- **配置迁移接口**：新增用户配置 schema 版本与迁移入口，并预留配置导出 / 导入接口，便于后续做用户配置备份。
- **卸载清理选项**：Windows 卸载器新增可选项，允许用户在卸载时删除本地用户数据、API Key 和更新缓存。

### 变更

- AI 队列前端只为 running 中的 Job 保持 SSE 通道，并限制活跃 EventSource 数量，降低批量加入 Job 时的连接压力。
- 桌面更新服务在打包环境中自动检查更新，开发环境保留更新检测能力但不执行真实安装链路。
- 更新弹窗支持 Markdown / HTML / 纯文本 Release Notes 渲染，并在安装前检查是否仍有运行中的 AI 队列任务。
- 默认搜索配置启用 Tavily 和自动域名，默认模型阶段改为 DeepSeek `deepseek-v4-flash[1m]` / `deepseek-v4-pro[1m]`。
- 版本发布约定统一为裸 semver tag，例如 `2.0.1`，并显式关闭 electron-builder 的隐式发布行为。

### 修复

- 修复当前版本与最新版本相同仍提示“发现新版本”的问题。
- 修复切换到 WordPreview 等页面后自定义标题栏消失、Back to List 无响应的问题。
- 修复 Settings/About 页面离开后更新按钮和查看更新细则失效的问题。
- 修复批量 Anki 导出中 Duplicate Overwrite 无效的问题，重复卡片可按真实 note/card 信息定位并覆盖。
- 修复 Anki duplicate 查询过度依赖目标 deck 的问题，改为按 note model 和字段匹配并回读实际卡组。
- 修复 AI 生成流水线中工具调用结果、诊断信息和 YAML 降级处理的若干稳定性问题。

### 文档

- 新增桌面自动更新 ADR，记录 electron-updater、GitHub Release assets 和 update feed 的设计约束。
- 新增 release notes 目录说明，规范按版本号保存 GitHub Release 内容来源。
- 更新项目上下文文档，记录默认配置、配置迁移、桌面更新和发布流程等术语。

## [2.0.2] - 2026-05-26

### 新增

- **Basic Format Fix**：新增确定性的 YAML 格式修复与诊断流程，保存、Workset 保存和 Fill Editor 前都会复用同一条校验路径。
- **Word Editor 一键修复**：编辑器新增 Repair 操作，可手动触发 Basic Format Fix，并在安全修复后回填修复后的 YAML。
- **重复 YAML 键诊断**：YAML parse error 现在能识别 duplicated mapping key，提示重复键名、首次出现行和重复出现行，便于手动修复。
- **工作集用户评分**：Workset 支持用户评分覆盖 AI 审核分，Workset Improve 使用有效评分判断是否需要继续修复。

### 变更

- Fill Editor 会先执行 Basic Format Fix；如果只能部分修复，会在弹窗说明后把可编辑 YAML 填入 Word Editor 供用户继续手动处理。
- Word Editor 实时校验改为 strict current YAML validation，只检查当前可见文本，不自动运行 Basic Format Fix。
- Word Editor 校验增加 Checking YAML 状态，并统一通过输入监听捕捉粘贴、Tab 缩进、Fill Editor 和程序化写入。
- Word Editor 行号列按 textarea 的实际软换行高度动态测量，长 YAML 行折行后行号仍与可见文本对齐。
- AI 生成队列抽出共享 QueueTable，Active Queue、Job History 和 Today Workset 使用更一致的表格布局、展开状态和操作列。

### 修复

- 修复重复生成时错误继承旧 notes 的问题，重生成任务只沿用应继承的上下文。
- 修复 Workset Improve 批量操作中按钮文案和低分筛选行为不一致的问题。
- 修复 YAML 字段层级错位时 Fill Editor 无法把可手动修复的 YAML 填回编辑器的问题。
- 修复 Word Editor 行号与软换行文本错位，避免用户按错误视觉行修缩进而造成 YAML 层级错误。
- 修复 Format Fix 遇到重复 section 时可能覆盖根区块的风险；现在只报告阻塞诊断，不静默合并或丢弃内容。

### 文档

- 更新项目上下文文档，记录 Format Fix、Format Diagnostic、Section Promotion、Duplicate Section、Workset Improve 等术语和边界。
- 新增 Workset Improve ADR，记录有效评分与工作集修复选择的决策。
- 更新队列表格相关文档和英文创意提示词说明。

### 移除

- 移除过期的 structured JSON 创意阶段 ADR，避免与当前 YAML 流程冲突。

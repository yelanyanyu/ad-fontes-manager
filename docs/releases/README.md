# Release Notes

Create one Markdown file per desktop release, named with the bare package version:

```text
docs/releases/2.0.1.md
```

The desktop release workflow uses this file as the GitHub Release body. Tags should also use
bare versions, such as `2.0.1`, not `v2.0.1`.

Suggested format:

```md
## [2.0.1] - 2026-05-23

### 新增

- ...

### 修复

- ...

### 注意

- ...
```
